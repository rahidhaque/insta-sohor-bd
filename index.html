In JavaScript, everything happens inside execution context where it contains a big box. Inside the box there are two components which are memory and code. Inside the memory component or variable environment, where it stores all the variables as key value pair and functions. Then there is a code component or thread of execution, where the code executes single line at a time which means JavaScript is a synchronous single threaded language or interpreted language. It means code cannot move down if the top of the code doesn’t execute. In the browser it helps to run the code with help of “V8” JavaScript engine. In the engine, first of all parser is used where it runs when every-time we run any JavaScript program. The main job of the parser is to check any error and if found it stop the program execution. Secondly it executes a tree where it contains how compiler will execute the code in detail steps. Thirdly, V8 engine will convert into code where machine can understand. Finally, the machine code converted JavaScript will be ready for deploy for execution.

Local storage and session storage works basically the same way in storing data but there are some differences. Local Storage can be accessible from any window but in session storage it can only be accessed from same tab. So for example when there is a multiple google chrome tabs local storage will be available on all tabs but session storage will not and it is unique per tab. Local storage will be even available when we close these google chrome tabs but if the user closes any tabs that he/she opened session storage will not be available and it will expire. Local storage will be available forever unless user ends up deleting it manually or through any code. In Local storage there might be some security issues like against cyber-attacks it is vulnerable and it is not effective. But session storage is somewhat more secured in these type of attacks than local storage.
